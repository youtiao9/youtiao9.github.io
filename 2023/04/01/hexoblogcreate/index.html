10分钟免费部署个人博客网站

更新日期:   2023-04-02

前言：在之前，看到大家的个人博客网站，我总是会很羡慕，总想着哪天自己也能搭建一个属于自己的网站。而今天，我将用hexo进行演示，如何花费10分钟搭建一个个人的博客网站

1. 安装Node.js&Git

这些软件是部署博客的基础，是最基本的步骤之一。

点击此处安装node.js，跳转后选择适合你的版本。

点击此处安装Git，跳转后选择适合你的版本。

下载完毕后，在安装程序内只需要一直[下一步]就行了

接下来，按住[Win]+[R]并输入cmd回车，打开cmd后分别输入以下指令：

node -v
npm -v
git -v

如果均显示版本号即为成功。

2. 安装Hexo

这是博客搭建的第二步，也是最基本的步骤之二。

点击此处进入hexo官网（或者直接点击https://hexo.io/）

在任意位置创建新的文件夹

接着，打开文件夹，在地址栏输入cmd并回车，或右键文件夹打开Git Bash Here

在终端内分别输入并回车

  1. npm install hexo-cli -g # 安装hexo

  2. hexo init blog # 创建blog文件夹

  3. cd blog # 进入博客文件夹

  4. npm install # 安装npm，如果不行可以尝试更换淘宝镜像源（下一条指令）

npm install -g cnpm --registry =https://registry.npm.taobao.org # 安装npm（淘宝镜像源）

  1. npm install hexo-deployer-git --save # 安装自动部署工具
  2. hexo server # 可简写为hexo s，运行hexo本地网站

3. 部署Hexo

在创建博客之后，该怎么才能让他人所知道与访问呢？

这便是最基本的步骤之三。

部署博客一般分为两种，第一种是使用Github的Github Pages，第二种是使用码云的Gitee Pages。这两种均是完全免费的博客部署与托管平台。

3-1. 通过Github部署

使用Github部署的优点

  • 无需实名
  • 安全保障

缺点则是访问速度非常慢，且需要魔法上网（自行解决）

那么，就开始部署吧。

进入Github，选择sign up或sign in进行登录/注册

登录成功后，选择屏幕右上方+号中的New epositories

在新建仓库界面里：

将仓库名设置为username.github.io（必须）

将Add a README flie打勾（这将会创建main分支）

描述可自定或不写。

接着点击

成功创建个人博客仓库~！

接下来，进入仓库内

这便是我们的博客地址(username.github.io)。

在任意位置打开cmd或Git Bash Here

输入

git config --global user.name "注册Github的用户名"
git config --global user.email "注册Github的邮箱"

接着输入

ssh-keygen -t rsa -C "你注册Github邮箱"

敲4次回车

然后进入C:\Users\你的Win用户名\.ssh\

进入后，打开id_rsa.pub文件

将文件内的ssh密钥全部复制。

将ssh密钥复制进框内

添加成功后，在任意位置打开cmd/Git Bash Here

输入ssh -T git@github.com

回车后输入yes（如果有）

返回以下信息

成功！

3-2. 通过Gitee部署

通过Gitee部署的优点：

  • 访问快速
  • 中文操作

缺点是上传的代码会受到监管，请注意言辞

打开Gitee，登录/注册后，进入设置

进行实名认证并等待审核完毕。

新建仓库

将仓库按照如下格式创建

请记住，新建仓库时：

一定要将仓库名称与路径设置为用户名

一定要选择开源（如果实名认证暂未通过，可以待通过后再在仓库配置中开源）

一定要打勾“初始化仓库”，将语言改为JavaScript

新建仓库后，默认分支为master

接下来，会进入这个页面

复制“全局设置”里的指令，在cmd/Git Bash Here打开并回车。

然后，输入

ssh-keygen -t rsa -C "你注册Gitee邮箱"

敲4次回车

然后进入C:\Users\你的Win用户名\.ssh\

进入后，打开id_rsa.pub文件

将文件内的ssh密钥全部复制。

将ssh密钥复制进框内

添加成功后，在任意位置打开cmd/Git Bash Here

输入ssh -T git@gitee.com

回车后输入yes（如果有）

返回以下信息

成功！

4. 博客文件的修改

基础的设置已完成，接下来是个性化的修改。

打开博客根目录的_config.yml文件

推荐使用微软出品的VS Code编辑器，点击下载

以下是网站的自定义部分

拉到最底部，是网站的重要配置部分

更改为

deploy:
  type: git
  repository: 
  branch:  # Github用户请更改为main
  # Gitee用户请将'branch:'这一行删掉

注意：后三个冒号后一定要加空格再到值

其中，repository部分的值请复制以下值

Github:

Gitee:

复制后，粘贴到repository: 后（一定要在冒号后添加空格）

接着，你可继续在_config.yml文件内编辑。编辑完成后记得[CTRL]+[S]保存

随后，在..\blog\文件夹内，打开cmd或Git Bash Here

先输入hexo s测试，并打开内网网站

如测试无误，即可输入[CTRL]+[C]结束网站

然后输入hexo g生成文件，hexo d部署至仓库

可直接输入hexo g -d生成并部署

此时，打开仓库

已经存在了网站文件！

稍后打开：

Github用户：https://username.github.io/

Gitee用户：

即可看到你生成的网站了~！

5. Hexo基本操作

既然已经成功部署了属于自己的博客，那么应该如何去发表呢？

这便是第五步会涉及的内容。

5-1 .发布第一篇博文

在hexo根目录下，打开终端。

输入hexo new 博文名字

（在未安装相关插件时，请一定使用小写字母！使用大写字母可能会导致部署后网站错误）
